# Expo Easy Passkey Context

## Glossary

### Relying Party

The service that owns the passkey relationship. Its RP ID is the domain used by `rp.id`, `rpId`, platform association files, and server verification.

### Ceremony

A WebAuthn registration or authentication operation. The Expo module runs the native ceremony and returns WebAuthn JSON to the app.

### Challenge

A fresh, random value generated by the backend for one ceremony. Challenges are single use and should expire quickly.

### Passkey Credential

The server-side record created after registration verification succeeds. It stores the credential ID, public key, sign counter, transports, device type, and backup state needed for later authentication verification.

### Demo Session

The placeholder session response returned by `apps/example-backend` after authentication verification succeeds. It demonstrates where a real product would create an app session.

### Demo Store

The concurrency-safe in-memory storage used by `apps/example-backend` for expiring ceremony records and passkey credentials. It safely isolates overlapping ceremonies within one process, but it is not production persistence and is not durable or coordinated across Vercel cold starts, multi-instance routing, or redeploys.
